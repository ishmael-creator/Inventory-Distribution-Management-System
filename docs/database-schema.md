# PostgreSQL Schema Design

The executable schema is in:

```text
backend/alembic/versions/0001_initial_schema.py
```

## Design Rules

- Primary keys are UUIDs generated by PostgreSQL `gen_random_uuid()`.
- Status values use PostgreSQL enums.
- Inventory balances are protected derived state.
- `inventory_transactions` is append-only application history.
- Balance changes and ledger writes must happen inside one database transaction.
- Audit logs are written for important workflow and administration actions.

## Core Tables

| Table | Purpose |
| --- | --- |
| `roles` | RBAC role codes and permission lists |
| `users` | Authenticated users assigned to roles |
| `products` | Product catalog, including ICS Klik, ICS Singapore, and EPC |
| `product_batches` | Manufacturing batches created by manufacturers |
| `warehouses` | Warehouse locations that receive stock |
| `hubs` | Hub locations attached to warehouses |
| `agents` | Agent records tied to user accounts |
| `inventory_balances` | Current stock by product and location |
| `inventory_transactions` | Immutable transaction ledger |
| `stock_movements` | Movement workflow history |
| `allocation_requests` | Distribution team requests |
| `dispatch_orders` | Dispatch workflow records |
| `receipts` | Receipt confirmation records |
| `audit_logs` | User/system action history |
| `notifications` | User notification inbox |

## Location Pattern

Inventory location is modeled with:

```text
location_type enum
location_id uuid
```

This keeps the ledger flexible across manufacturers, warehouses, hubs, and agents without duplicating transaction tables for each location type.

## Important Constraints

- Product SKU is unique.
- Product batch number is unique per product.
- Inventory balance is unique per product and location.
- Quantities must be positive for movement records.
- Balances cannot be negative.
- Reserved quantity cannot exceed total quantity.

## Phase 1 Ledger Events

| Event | From | To | Transaction Type |
| --- | --- | --- | --- |
| Production batch created | none | manufacturer | `PRODUCTION` |
| Warehouse receipt confirmed | manufacturer | warehouse | `WAREHOUSE_RECEIPT` |

Later phases add allocation, dispatch, hub receipt, agent receipt, transfers, and reversal workflows.
